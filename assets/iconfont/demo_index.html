Unicode Reference

Unicode is the most basic way to use fonts on web pages. Features:

  • Best compatibility, supports IE6+ and all modern browsers.
  • You can dynamically adjust icon size, color, etc. like fonts.
  • But since it is a font, it does not support multi-color icons.

Note: The new iconfont supports multi-color icons, but they cannot be used in Unicode mode. If needed, use the symbol method.

Steps to use Unicode:

Step 1: Copy the generated @font-face


@font-face {
  font-family: 'iconfont';
  src: url('iconfont.eot');
  src: url('iconfont.eot?#iefix') format('embedded-opentype'),
      url('iconfont.woff2') format('woff2'),
      url('iconfont.woff') format('woff'),
      url('iconfont.ttf') format('truetype'),
      url('iconfont.svg#iconfont') format('svg');
}

Step 2: Define iconfont style


.iconfont {
  font-family: "iconfont" !important;
  font-size: 16px;
  font-style: normal;
}

Font Class Reference

Font-class is a variation of Unicode that makes icons easier to understand.

  • Good compatibility (IE8+).
  • More readable than Unicode.
  • You only need to change class name to replace icons.
  • Still does not support multi-color icons.

Symbol Reference

This is a newer method and the recommended way.

  • Supports multi-color icons.
  • You can change style using CSS.
  • Supports IE9+ and modern browsers.
  • SVG rendering performance is average.
